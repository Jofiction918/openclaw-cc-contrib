# extract-memories — 对话记忆提炼

每次对话结束后自动分析本轮完整上下文，提炼所有值得持久化的信息，写入 `memory/YYYY-MM-DD.md`，让人工智能在下一次对话时拥有完整记忆，无需用户重复说明背景。

## 何时使用

- **自动触发**：每次对话结束（模型产出最终回复后）自动后台执行
- 手动触发：`/extract-memories` 手动触发一次提炼

## 执行步骤

1. **读取上下文**：获取本轮对话的完整消息历史（sessions_history 或直接引用当前上下文）
2. **全面扫描**：不只关注"明确说出口"的决策，还要挖掘：
   - 用户没有明说但隐含的偏好（语气、格式、沟通风格）
   - 对话中暴露的项目结构和代码位置
   - 涉及的人名、工具名、术语、路径
   - 讨论过程中暴露的空白知识（用户承认不知道的事）
   - 争论的焦点和最终谁说服了谁
3. **多维度分类**：按本轮对话实际出现的 topic 生成类目（类目不固定）
4. **写入 memory/YYYY-MM-DD.md**（APPEND 模式，不覆盖已有内容）
5. **通知主会话**：按标准格式输出提炼结果

## 输出模板（固定格式，每次必须遵循）

```markdown
## {{YYYY-MM-DD}} 记忆提炼

**提炼结果**：N条
**写入位置**：`memory/{{YYYY-MM-DD}}.md`

---

### ▸ 决策
- 内容1（来源依据：具体对话/文件名/配置）
- 内容2

### ▸ 偏好
- 内容1（表达场景：在XX情境下）
- 内容2

### ▸ 项目约定
- 内容1
- 内容2

### ▸ 技术方案
- 内容1
- 内容2

### ▸ 开放问题
- 内容1（阻塞程度：🔴严重/🟡中等/🟢轻微 → 下一步：XX）
- 内容2

### ▸ 参考资料
- 内容1（URL/路径：XX → 备注：XX）
- 内容2

### ▸ [自由类目A]
- 内容1
- 内容2
```

## 类目生成规则

- **类目不固定**，根据本轮对话实际出现的 topic 自由生成
- 常见基础类目：`决策` / `偏好` / `项目约定` / `技术方案` / `开放问题` / `参考资料`
- 这些基础类目优先使用，如果没有相关内容则跳过该类目
- 完全按本轮实际 topic 生成新类目（如 `产品` / `人脉` / `风险` / `财务` 等）
- 类目数量不限，对话短就少类目，对话长就多类目
- 每个分类若无内容，写 `（暂无）` 或 `（空）`，不能留空

## 主会话通知格式（固定格式）

提炼完成后，在主会话按以下格式输出（**纯 Markdown，不使用 ASCII 艺术字符**）：

```markdown
## {{YYYY-MM-DD}} 记忆提炼

**提炼结果**：N条
**写入位置**：`memory/{{YYYY-MM-DD}}.md`

---

### ▸ 决策
- 内容1
- 内容2
### ▸ 偏好
- 内容1
- 内容2
### ▸ [自由类目]
- 内容1
```

## 写入规范

- **APPEND 模式**：追加到 `memory/{{date}}.md` 末尾，不覆盖已有内容
- 文件不存在则自动创建
- 日期格式：`YYYY-MM-DD`
- 每条内容一句话，直接了当，不加多余描述

## 权限要求

- `FileRead`：读取对话上下文
- `FileWrite` / `FileEdit`：写入 memory 目录
- `sessions_spawn`：（可选）fork 子 Agent 并行处理大量历史

## 触发词

- 自动：对话结束（skill 内置触发逻辑）
- 手动：`/extract-memories`
