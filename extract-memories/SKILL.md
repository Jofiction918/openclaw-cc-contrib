# extract-memories — 对话记忆提炼

每次对话结束后自动分析本轮完整上下文，提炼所有值得持久化的信息，写入 `memory/YYYY-MM-DD.md`，让人工智能在下一次对话时拥有完整记忆，无需用户重复说明背景。

## 何时使用

- **自动触发**：每次对话结束（模型产出最终回复后）自动后台执行
- 手动触发：`/extract-memories` 手动触发一次提炼

## 执行步骤

1. **读取上下文**：获取本轮对话的完整消息历史（sessions_history 或直接引用当前上下文）
2. **全面扫描**：不只关注"明确说出口"的决策，还要挖掘：
   - 用户没有明说但隐含的偏好（语气、格式、沟通风格）
   - 对话中暴露的项目结构和代码位置
   - 涉及的人名、工具名、术语、路径
   - 讨论过程中暴露的空白知识（用户承认不知道的事）
   - 争论的焦点和最终谁说服了谁
3. **多维度分类**：按下方模板的六个维度组织信息
4. **写入 memory/YYYY-MM-DD.md**（APPEND 模式，不覆盖已有内容）
5. **通知主会话**：输出一条简洁确认（提炼了多少条，涉及哪些维度）

## 输出模板

写入文件时严格遵循以下格式，每条内容直接写，一目了然，不加多余的笼统标签：

```markdown
### {{YYYY-MM-DD}} 记忆提炼 · {{一句话概括本轮核心主题}}

**提炼结果**：共 N 条（决策× 条 / 偏好× 条 / 约定× 条 / 技术× 条 / 问题× 条 / 参考× 条）
**写入位置**：memory/{{YYYY-MM-DD}}.md

---

### 决策
- 内容1（来源依据：具体对话/文件名/配置）
- 内容2

### 偏好
- 内容1（表达场景：在XX情境下）
- 内容2

### 项目约定
- 内容1（涉及范围：XX）
- 内容2

### 技术方案
- 内容1（问题：XX → 方案：XX → 验证：XX）
- 内容2

### 开放问题
- 内容1（阻塞程度：🔴严重/🟡中等/🟢轻微 → 下一步：XX）
- 内容2

### 参考资料
- 内容1（URL/路径：XX → 备注：XX）
- 内容2
```

## 分类标准说明

**决策**：用户明确拍板的事项，包括被否决的替代方案及其原因（方便以后避免重蹈覆辙）。每条格式：决策内容 + 括号注明来源依据。

**偏好**：用户表达过的习惯和风格，包括隐含偏好（用户没明说但从行为推断）。每条格式：偏好内容 + 括号注明在什么场景下表达。

**项目约定**：目录结构、Git规范、代码风格、命名规范、流程规范等。每条格式：约定内容 + 括号注明涉及范围。

**技术方案**：问题→解决方案→验证结果的完整链路，附具体配置值/版本号/关键代码。每条格式：问题→方案→验证。

**开放问题**：已知但未解决、需要用户回答、依赖外部的问题。每条格式：问题描述 + 阻塞程度 + 建议下一步。

**参考资料**：URL、文档、论文、文件路径。每条格式：资料描述 + URL/路径 + 为什么值得保存。

## 写入规范

- **APPEND 模式**：追加到 `memory/{{date}}.md` 末尾，不覆盖已有内容
- **文件不存在**：自动创建 `memory/` 目录和文件
- **日期格式**：`YYYY-MM-DD`，使用当天日期
- **标题**：`{{YYYY-MM-DD}} 记忆提炼 · {{一句话概括}}`

## 主会话通知格式

提炼完成后，在主会话输出一条确认：

```
提炼结果：N条
写入位置：memory/{{date}}.md
---
### 决策
- 内容1
- 内容2
### 偏好
- 内容1
- 内容2
...
```

## 权限要求

- `FileRead`：读取对话上下文
- `FileWrite` / `FileEdit`：写入 memory 目录
- `sessions_spawn`：（可选）fork 子 Agent 并行处理大量历史

## 触发词

- 自动：对话结束（skill 内置触发逻辑）
- 手动：`/extract-memories`
