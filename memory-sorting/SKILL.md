# memory-sorting v2.0.0 — 衣柜式记忆整理

定期整理 `MEMORY.md` 和 `topics/` 目录，检测重复、过时、冲突内容，生成提案等你批准，批准后执行修改。

## 何时使用

- **手动触发**：`/memory-sorting`
- 建议频率：每周一次，或任何你觉得记忆库开始乱的时候

## 两种模式

### 快速扫描（不修改文件）

只读扫描，发现问题后立即生成提案，用户批准再执行。

### 完整扫描（需要写入权限）

用户批准后执行，执行结果写入文件并输出报告。

---

## 执行步骤

### Step 1 — 全量扫描

扫描范围：
```
MEMORY.md                          ← 纯索引（每行一个指针）
topics/*.md                         ← 所有 topic 记忆文件
memory/*.md 或 memory/logs/        ← daily 日记（作为时间线参考）
```

标记每条记忆的：
- **来源文件**（MEMORY.md 或某个 topic）
- **出现次数**（判断是否重复）
- **最后更新时间**（判断是否过时）
- **冲突**（同一问题不同文件说法矛盾）

### Step 2 — 问题检测

| 类型 | 检测逻辑 | 标记 |
|------|---------|------|
| 🔁 重复 | 同一内容在 ≥2 个 topic 文件中出现 | 重复 |
| ⏰ 过时 | topic 内容被 daily 最新结论推翻；或相对日期存在超过 30 天 | 过时 |
| ⚡ 冲突 | 同一问题两个 topic 说法矛盾 | 冲突 |
| ❓ 孤儿 | topic 文件从未被 MEMORY.md 引用 | 孤儿 |
| 📜 碎片化 | 同一主题分散在 ≥3 个文件中 | 碎片 |
| 📏 超限 | MEMORY.md 超过 200 行或 25KB | 超限 |

### Step 3 — 生成提案

**不修改任何文件**，只输出提案等待批准。

---

- 主题聚合程度：{{N}}/25（碎片化/重复检测）

## Step 4 — 执行（收到批准后）

按顺序执行：
1. 执行所有 PO（先更新 topic，再更新 MEMORY.md）
2. 执行所有整理清理
3. 验证修改后文件可读
4. 删除孤儿 topic 文件
5. 输出执行报告

## Step 5 — 执行报告

```markdown
## 记忆整理执行完毕

**执行时间**：{{YYYY-MM-DD HH:MM}}
**实际执行**：{{M}}条（与提案一致 / 有调整）

| 操作 | 数量 | 详情 |
|------|------|------|
| 必须做(PO) | {{N}}条 | [列出] |
| 整理清理 | {{N}}条 | [列出] |
| 跳过 | {{N}}条 | [列出原因] |

**MEMORY.md**：{{N}}行 · 大小 {{N}}KB
**topics/**：{{N}}个文件 · 新增 {{M}}个 · 删除 {{K}}个
```

## 检测例外（不会标记为问题）

- AI 资讯来源列表（不同文件出现相同 URL 是正常现象）
- 同一个人的联系方式出现在不同 context（不视为重复）
- 时间戳不同但内容相似的 TODO（视为不同时间点的独立记录，不合并）
