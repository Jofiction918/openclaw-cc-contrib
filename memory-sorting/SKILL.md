# memory-sorting — 像整理衣柜一样整理记忆

整理 `MEMORY.md` 和 `memory/*.md`，自动检测：
- **重复内容**：已经在 `MEMORY.md` 的条目，自动提示删除重复
- **过时内容**：旧结论被新对话推翻，自动提示更新
- **冲突内容**：不同记忆说法矛盾，提示解决
- **归属模糊**：分不清是项目规范还是个人偏好，提问用户确认

整理完成后输出提案报告，等待用户审批，批准后再执行修改。

## 何时使用

- 手动触发：`/memory-review`，定期整理记忆
- 当记忆文件超过一定行数自动建议整理

## 工作流程

### 1. 读取所有记忆文件
- 读取 `MEMORY.md`（项目级长记忆）
- 扫描 `memory/*.md` 所有每日提炼文件
- 汇总所有记忆条目

### 2. 分类检测
- **重复**：同一条信息出现在多个文件
- **过时**：新结论和旧结论矛盾
- **冲突**：同一问题有不同结论
- **归属**：分不清是项目规范还是个人偏好

### 3. 生成提案报告
按「 promotions / cleanup / ambiguous」分类，逐条说明：
- 要移动什么 → 从哪里 → 到哪里
- 要删除什么 → 为什么
- 要修改什么 → 旧内容 → 新内容
- 需要用户确认什么 → 归属问题

### 4. 等待审批
用户批准后，按提案执行文件修改

### 5. 完成输出
修改完成后，输出修改统计：移动 × 条，删除 × 条，修改 × 条，确认 × 条

## 权限要求
- 需要 `FileRead`、`FileEdit`、`FileWrite` 权限
- 需要 `sessions_spawn` 提案分析（可选）
