# dream-rem — 睡梦式记忆深度整合

定时深度整合记忆（默认 ≥24小时 + ≥5个新会话），将分散的 daily notes 合并到主题索引中，删除过时内容，保持记忆系统长新。

## 何时使用

- **自动触发**：满足 `≥24小时` + `≥5个新会话` 条件时后台自动运行
- **手动触发**：`/dream-rem` 强制触发一次深度整合

## 前置条件

- `extract-memories` 正常工作（daily notes 是整合的原材料）
- 写锁 `.consolidate-lock` 不存在（防止多设备同时整理冲突）

## 执行流程

1. **扫描**：读取 MEMORY.md、memory/ENTRYPATH.md（若存在）、所有 daily 文件
2. **诊断**：对照"当前记忆结构诊断"逐项检查
3. **生成提案**：按"必须做(PO) → 整理清理"两级分类，每条含现状 + 目标
4. **等待批准**：输出提案，用户批准后再执行
5. **执行整理**：更新 MEMORY.md / 新建主题文件 / 归档过时文件 / 重建 ENTRYPATH.md
6. **清理锁文件**：删除 `.consolidate-lock`

## 格式说明

- **本次变化概览**：对比上次整合时的状态，快速判断记忆系统是变好还是变坏
- **记忆健康度评分**：综合评分 + 四个维度各25分，一眼判断记忆系统整体状态
- **重要条目提取**：本次整合中发现的最有价值的3-5条记忆，不需要多，但要精
- **待确认冲突列表**：无法自动判断、需要用户拍板的问题（版本冲突/归属争议/待办定性等），每条列A/B方案和我的建议
- **ENTRYPATH.md快照**：整合后的完整索引内容，方便人工核对是否正确
- **下次整合预计**：基于当前时间和触发条件，给出下次自动整合的预计时间
- 每条行动必须有：`现状`（当前是什么）和`目标`（改成什么样），不能只写行动名称
- 提案按紧急程度排序：PO 先于 整理清理

## 分布式锁

- 获取锁：创建 `.consolidate-lock`（写入主机名+时间戳）
- 锁存在：退出，报告"另有整理在进行"
- 释放锁：整理完成或中断时删除 `.consolidate-lock`

## 权限要求

- `FileRead`：读取所有 memory 文件
- `FileWrite` / `FileEdit`：修改 MEMORY.md、daily 文件、ENTRYPATH.md
- `sessions_spawn`：（可选）fork 子 Agent 并行分析

## 触发词

- 自动：≥24小时未整合 + ≥5个新会话（skill 内置定时检查）
- 手动：`/dream-rem`
